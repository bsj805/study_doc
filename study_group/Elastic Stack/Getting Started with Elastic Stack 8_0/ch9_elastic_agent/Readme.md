# 새 데이터 소스 온보딩 


비트 에이전트를 사용하면, 여러 데이터 수집은 가능한데, 계속해서 데이터를 추가하게 된다면, 매번 비츠 구성을 해야한다. 
- Elastic agent와 Fleet 서버로 해결하자. 

### 단일 에이전트로 통합 데이터 수집 

Elastic 에이전트는 데이터 수집기능을 단일 에이전트로 통합 -> 여러 서버에 배포 관리 업그레이드 안해도 됨.
- Filebeat, HeartBeat 등을 갖고있는게 Elastic Agent. 

Fleet UI는 키바나 어플리케이션으로, 에이전트 구성해서 온보딩하고, 전체 에이전트를 통제할 수 있는 UI. 

Fleet 서버는 엘라스틱 에이전트 전체를 연결해 에이전트 정책, 업데이트, 관리 명령을 검색할 수 있는 백엔드 컴포넌트 
- 에이전트는 ES에서 정책을 검색해서 에이전트에 자동으로 적용한다.
