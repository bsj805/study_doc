# 개발 독립성

업무 규칙이 UI를 알지 못하면 UI에 중점을 둔 팀은 업무 규칙에 중점을 둔 팀에 영향을 줄 수 없다.

유즈케이스도 결합이 분리되면 addOrder 유스케이스에 중점을 둔 팀이 deleteOrder 유즈케이스에 중점을 둔 팀에 개입할 가능성은 없다

- 물론 이는 AOP처럼 개별 유즈케이스마다 코드가 추가되는 방식일 때에
